# OpenClaw Launcher - UI/UX 重构需求文档 (UI PRD V2)

## 1. 目标与定位 (Goals & Positioning)
在完成 Phase 3.5 的核心三栏（仪表盘/AI引擎/设置中心）与 Premium Dark Mode（极简深色高级皮肤）基础搭建后，Phase 3.6 (V2) 的核心目标是**统一二级页面的视觉语言**，并引入**微动效 (Micro-interactions)**，彻底消除廉价感，在一加一减中打造出具有 Vercel / Linear 级别克制而灵动的数字体验。

## 2. 核心痛点与优化点 (Pain Points & Solutions)

### 2.1 全局启动状态感知缺失
*   **当前问题**：点击“初始化并启动”后，后台服务拉起需要几秒钟，在此期间 UI 仅有按钮文字变化，用户感知极弱，容易误以为卡顿。
*   **V2 目标**：
    *   引入全局或区域级的丝滑加载动画（如 `lucide-react` 的 `Loader2` 或 CSS 呼吸光环）。
    *   明确的“核心引擎唤醒中...”等加载文案提示。

### 2.2 二级页面与弹窗 (Modal) 色彩违和
*   **当前问题**：
    *   主色调已改为极简黑/透，但底层配置弹框、Tab 选项卡（如“免费注册”、“自定义中转站”）激活时仍是大面积亮蓝色。
    *   模型选择按钮、保存完成的打勾按钮使用了突兀的亮绿色。
    *   弹窗内的表单输入框排版拥挤，特别是“自定义中转站”单薄且局促。
*   **V2 目标**：
    *   **色彩降噪**：彻底移除突兀的蓝/绿高亮。激活状态 (Active) 改为细腻的半透明白/浅灰背景，配合微光边框（参考左侧设侧边栏）。
    *   **按钮重构**：主操作（如保存）采用黑底白字或高反差的 `btn-primary` 样式，次要操作为描边或透明。移除自带廉价感的原生 Emoji 勾选。
    *   **版式呼吸感**：适当拉宽“自定义中转站”及表单类 Modal 的尺寸。输入框背景使用极低透明度 (`rgba(255,255,255,0.05)`)，获取焦点时仅改变边框色度。

### 2.3 图标风格割裂与缺乏动效
*   **当前问题**：
    *   模型页标题仍在使用 Emoji `🤖`，与主导航的 `lucide` 线条 Icon 不搭。
    *   页面切换、弹窗呼出过于生硬，缺乏物理世界的缓冲质感。
*   **V2 目标**：
    *   **Icon 统一**：全面使用 `lucide-react`。如将 `🤖` 替换为 `Cpu` 或 `Network`。
    *   **引入高级动效库 (`framer-motion`)**：
        *   为 Modal 弹框添加带物理弹簧属性 (Spring) 的缩放/渐隐出场。
        *   为主内容区域切换添加平滑的透明度过渡。
        *   按钮悬停 (Hover) 时加入丝滑的缩放反弹。

## 3. 具体执行清单 (Action Items for Phase 3.6)

1.  [ ] **依赖引入**: 确保项目中集成并可用 `framer-motion`, `clsx`, `tailwind-merge` 等 UI 增强库。
2.  [ ] **全局加载状态**: 在 Dashboard 的 Hero 区域，替换启动时的生硬文本，加入平滑旋转的 loading icon 与过渡字样。
3.  [ ] **CSS 色彩规范洗牌**: 搜索并清理 `App.css` 中所有造成“绿/蓝”等明度过高的高亮选择器（如 `.category-btn.active`, `.model-select-btn.active`，原设置表单等）。
4.  [ ] **Modal 结构层重写**: 
    *   重塑 Key Modal 的布局，扩宽 `maxWidth`。
    *   统一表单 Input 的样式。
    *   使用高反差风格重写 Save 按钮。
5.  [ ] **补齐 `lucide-react` 替换**: 扫描仍遗留非语义化 Emoji 的标题或文案并替换。
6.  [ ] **Motion 动效注入**: 使用 `<motion.div>` 包装关键 UI 节点（如主要 Tab 的切换区块、所有的 `modal-box`）。

## 4. 验收标准
1.  **色彩一致性**：整个应用没有刺眼原色，遵循黑/白/深灰/透体系，仅在系统级错误/告警时允许少量极度克制的红/黄散点。
2.  **过渡流畅度**：从点击到弹窗，具备 60fps 级别的顺滑感输入和反馈，体验“无缝拉升”。
3.  **开发构建**：无 TypeScript 构建报错，Framer Motion 引入不造成明显的白屏阻塞。
