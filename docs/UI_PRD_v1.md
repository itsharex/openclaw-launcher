# OpenClaw Launcher - UI 重构与架构设计 PRD (Phase 3.5)

## 📌 1. 设计主旨 (Design Philosophy)
OpenClaw Launcher 致力于提供**「极客、极简、高级感 (Premium Dark)」**的体验。
核心目标是对标顶级开发者工具（如 Vercel Dashboard, Linear, Raycast），放弃传统、臃肿的面板设计，追求极致的信噪比。

## 📐 2. 视觉语言规范 (Design System)

*   **基础配色 (Colors)**：
    *   主背景：纯黑 (`#000000`) 或深色曜石 (`#09090b`)。
    *   卡片与浮层：低透明度白 (`rgba(255, 255, 255, 0.03)`) 搭配毛玻璃 (`backdrop-filter: blur(12px)`)。
    *   边框风格：极细且克制 (`border: 1px solid rgba(255, 255, 255, 0.08)`)。
    *   点缀色 (Accent)：高级紫 (`#8b5cf6`)、微蓝 (`#3b82f6`) 或绿（仅用于成功状态）。
*   **字体 (Typography)**：
    *   无衬线字体为主 (`Inter`, 系统默认中文字体)。
    *   日志、模型 ID、路径等数据展示强制使用等宽字体 (Monospace)。
    *   高对比度层级：主标题纯白，辅助信息弱灰。
*   **Icon 图标体系**：
    *   全面引入 **`lucide-react`**。
    *   **原则**：拒绝具象化（不用小机器人、聊天气泡），采用**抽象、几何、科技感**线框（如 `<Cpu />`, `<Activity />`, `<Hexagon />`）。
    *   配置极细线宽 (`strokeWidth={1.5}`)。
*   **窗口行为 (Window Handling)**：
    *   **默认尺寸**：`900px 宽 × 640px 高`，比例修长协调。
    *   **启动位置**：强制屏幕居中。
    *   **最小尺寸**：`min-width: 800px; min-height: 600px`。
*   **全局布局框架**：
    *   独立顶栏：悬浮毛玻璃导航条。
    *   主内容区：强制定宽水平居中 (`max-width: 860px`)，超大屏幕下内容不会割裂。

## 🗂️ 3. 三大核心产品架构 (The 3-Tab Structure)

基于**将「日常发车控制」与「系统底层配置」彻底分离**的黄金法则，应用精简为 3 个一级导航 Tab。

### Tab 1: ⚡️ 仪表盘 (Home / Dashboard)
**定位**：驾驶舱，最高频页面，极致干净。
*   **顶部视觉焦点**：呼吸状态指示灯（运行中 / 已停止）。
*   **操作中枢**：两颗大尺寸微交互胶囊按钮：「启动/停止服务」、「打开网页端」。
*   **环境快照 (只读)**：用一张极薄的毛玻璃卡片显示当前关键参数（当前模型 ID、端口、工作区）。
*   *注：坚决不在此层级放置 API Key 输入框等重度表单。只允许在已配好的模型间快速切换。*

### Tab 2: 🧠 AI 引擎库 (Providers / Engine)
**定位**：发动机总成，OpenClaw 的模型应用商店。
*   **视觉呈现**：展示各路大模型供应商（阿里云百炼、智谱、Minimax、OpenRouter 等）的并排网格卡片 (Grid Cards)。卡片上仅保留 Logo、名称与「免费/付费」角标。
*   **核心交互**：点击任意供应商卡片，**屏幕中央弹出带有深色毛玻璃背景的 Modal（悬浮面板）**。用户在 Modal 中填写 API Key 和指定默认模型。
*   *设计意图*：将繁杂的表单输入封装在遮罩层后，保持底层列表的极其清爽。

### Tab 3: ⚙️ 设置中心 (Settings)
**定位**：后台控制面板。
改用 **「左侧导航列表 + 右侧内容展示」** 的经典后台双列排版。拆分为三个子路由栏目：

#### 3.1 🎛️ 通用 (General)
*   **主题与外观**：选项卡（Dark 主推，后续拓展 Light/System）。
*   **系统集成**：开机自启动开关。
*   **文件系统**：默认工作区路径设定（展示当前路径，带修改按钮）。

#### 3.2 📄 日志诊断 (Diagnostics)
*   *注：将原一级 Tab 的日志降级至此，隐藏技术细节。*
*   **日志微视窗**：一个小巧的黑色代码框，仅滚动显示最新 20 行日志流，供极客用户排障快速查阅。
*   **核心功能**：下方放置一个巨型按钮 **[📦 一键导出诊断 ZIP 包]**，自动打包临时目录的 `.log` 文件供用户发送给支持团队下载。

#### 3.3 ℹ️ 关于平台 (About)
*   **视觉头部**：巨大且居中的 OpenClaw 标志与粗体名称。
*   **版本控制面板**：
    *   当前构建版本号（如 `v0.3.2`）。
    *   带有 `<RefreshCw />` 旋转动画的 **[🚀 检查更新]** 按钮。
*   **社区与支持资源**（采用 Linktree 样式的居中横向圆角按钮阵列）：
    *   `💬 加入交流群 / 联系作者` 
    *   `📘 关注微信公众号` (点击即中心弹出一个 Modal 展示二维码图片)
    *   `☕ 赞赏与支持` (点击弹出赞赏码)
    *   `⭐ 查看 GitHub 源码` (点击直接新标签页跳转)

## 🚀 4. 实施收敛路径 (Implementation Path)
1. **基础环境准备**：接入 `lucide-react`，更新 Tauri 窗口尺寸配置。
2. **样式重构**：大范围清理 `App.css`，注入全新的 CSS 变量与深色毛玻璃组件类。
3. **框架搭骨**：实现顶栏 Navbar 与最大宽度居中内容区。
4. **逐页精修**：
   - Phase A: 仪表盘去线框化。
   - Phase B: Provider 列表栅格化 + Modal 表单注入。
   - Phase C: 设置页的左侧边栏路由搭建与子组件组装。
