# 完整版 PRD: OpenClaw Launcher 桌面端产品需求说明书

*版本号: 1.0.0 (内部定稿版)*

## 一、 产品概述 (Executive Summary)

### 1.1 产品愿景
OpenClaw Launcher 致力于成为开源 AI Agent 生态中最简单、“傻瓜化”的桌面启动器。其核心使命是彻底消除非技术用户在安装、配置和运行 AI 框架时面临的“环境障碍”（Node.js、Git、终端报错），将其打包为一个“双击即用”的开箱产品，让每一个小白用户都能在一分钟内拥有自己的数字员工大脑。

### 1.2 核心用户群体与痛点
*   **目标用户:** 缺乏开发经验的小白用户、非科班的业务人员、对技术稍有了解但厌倦繁琐配置的极客。
*   **核心痛点解决策略:**
    *   **没有运行环境？** -> 底层悄悄下载。
    *   **国内网络超时？** -> 自动切换 npm 镜像源。
    *   **不知如何配置模型？** -> 自动写入官方免费 API 网关。
    *   **AI 没有场景？** -> 随包自带生成技能（skill_creator）和寻找技能（skill-lookup）的基础模块。

## 二、 系统架构与技术选型 (Architecture & Tech Stack)

### 2.1 技术矩阵
*   **后端进程管理(壳):** `Tauri v2` (基于 Rust)。选用理由：打包体积小（几十MB内，远胜 Electron）、内存占用低、原生系统资源调用稳健，不易被杀软列为高危脚本环境。
*   **前端交互(UI):** `React 18+` 或 `Vue 3` 配合 `Tailwind CSS` 构建现代化、大圆角、极简极客风界面。
*   **核心驱动(Engine):** `Node.js (Portable)`。**核心约束：绝不依赖用户系统全局环境变量！** 所有运行依赖一个下载或随包附带的局部 Node 二进制文件。

### 2.2 防杀软与权限提权策略 (UAC & AV Bypass)
为确保装机率（避免启动即卡死或遭系统拦截），必须严格遵守以下系统级规范：
1.  **用户空间隔离 (User-Space Sandboxing):** 所有的源码下载、运行环境包、工作库文件必须强制存放在 `AppData\Local\OpenClawLauncher` (Win) 或 `~/Library/Application Support/OpenClawLauncher` (Mac) 目录。规避系统盘（C盘）写保护，实现**免管理员权限运行**。
2.  **底层操作代码化 (API over Shell):** 严禁通过 `node child_process` 执行 `.ps1` (PowerShell) 或 `.bat` 这种容易被杀软判定为后门的脚本。解压ZIP、发网络请求、创建文件必须通过 Tauri (Rust) 原生代码或纯 Node.js api (`fs`, `axios`) 实现。
3.  **防火墙静默穿透:** 在使用 `Inno Setup` 等工具制作最终 `.exe` 固件安装包时，在安装完毕的终末环节（此时继承安装程序的提权状态），静默向系统防火墙写入本地 Node Web Server 端口的放行白名单，免除用户启动服务时的弹窗恐吓。
4.  **数字签名 (Code Signing):** 最终发版的二进制可执行文件必须申请并签入 EV (Extended Validation) 或标准系统代码签名证书，这是防 Windows SmartScreen 红色拦截框的唯一手段。

## 三、 产品核心业务流程图 (User Workflow)

此为主界面启动按钮背后的核心“黑盒流程”：

```mermaid
graph TD
    A([用户启动应用]) --> B{环境探针: 检查内置 Node 是否存在}
    B -- 暂无 --> C[静默下载对应架构 (x64/ARM) Portable Node.js 并释放至 AppData]
    B -- 已有 --> D{网络探针: 并发 Ping Github/NPM}
    C --> D
    D -- GitHub 超时 --> E[切换下载源: 走 OSS 镜像 ZIP 拉取 OpenClaw 源码]
    D -- 网络良好 --> F[拉取 OpenClaw 源码 ZIP 并解压]
    E --> F
    F --> G[执行内置 Node: `npm install --omit=dev`]
    G -- Install 失败 --> G1[自动切换 Taobao 镜像重试]
    G1 --> H[写入固定端口等基建 openclaw.json]
    G -- 成功 --> H
    H --> I[写入默认免费模型网关 models.json]
    I --> J[拷贝随包自带的 `skill_creator` 和 `skill-lookup`]
    J --> K([环境就绪：点亮 UI 启动按钮])
```

## 四、 具体功能模块需求说明

### 模块 A. 极简应用界面 (Frontend GUI)
**1. 启动视窗 (Initialization View)**
*   **交互:** 刚安装/首次双击打开应用时展示。
*   **内容:** 一个平滑滚动的进度条及加载文案（“正在为您搭建数字大脑环境 45%...”）。
*   **要求:** 背景纯黑或毛玻璃质感，突出科技感。禁止出现任何终端黑框报错代码。

**2. “AI 工作台”配置向导 (Workspace Setup)**
*   **触发时机:** 环境安装完成，第一次点击“Start”前。
*   **交互:** 弹出醒目对话框：“请为 OpenClaw 选择一个文件夹作为它的「物理办公区」，它产生的所有代码和文档都将存放在这里。”。提供一个默认选择（如桌面新建的 `OpenClaw Workplace`）。

**3. 主控看板 (Dashboard View)**
*   **巨型启停按钮 (Action Button):** 整个界面的视觉中心。
    *   状态 1: `Start` (主色调蓝/紫)，点击触发 `npm start`，显示 Loading 呼吸灯。
    *   状态 2: `Stop` (主色调红)，显示“✅ OpenClaw 运行中... `http://localhost:3000`”。
*   **快捷面板 (Quick Actions):**
    *   `[打开网页端 UI]`: 按钮，点击即调用系统默认浏览器打开本地 3000 端口（这才是真实与 AI 对话的地方）。
    *   `[打开办公桌]`: 按钮，调用系统资源管理器打开之前设置的 Workspace 目录。
*   **人话版日志区 (Humanized Log Pane):**
    *   过滤底层 `console.log` 的杂音（堆栈追踪等），只呈现 `Info`, `Warning`, `Error` 的提炼信息（如：“✅ 模型网关连通成功”、“🤖 AI 正在使用 skill_creator 思考中”等）。遇到致命错误（如端口 3000 被占用），日志需输出红色的修复建议文案提示。

### 模块 B. 后台托盘与常驻进程管理 (System Tray & Process Daemon)
*   **托盘隐藏:** 点击主界面的红叉 `X`，应用窗口隐藏，但不杀死 Node 进程，保证浏览器端的对话不中断，任务栏右下角保留应用图标。
*   **托盘右键菜单:** 提供 `[显示主窗口]`, `[重启 AI 大脑]`, `[彻底退出 (杀死后台并关软件)]` 功能。
*   **进程防多开:** 采用锁文件或互斥量 (Mutex) 阻止用户频繁双击图标导致的极度卡顿或端口抢占崩溃。

### 模块 C. 进阶设置面板 (Advanced Settings)
*   界面侧边提供一个齿轮图标进入此隐藏较深的区域。
*   **环境重置 (Nuke Core):** 一个标红的危险按钮，提示“当软件发生无法挽回的崩溃时使用”。功能：杀死 Node 进程，清空 AppData 中的代码和 Modules 缓存，触发下次启动时重新下载和初始化。这是“终极修复绝招”。
*   *(Phase 3预留)* **模型切换配置 (Model Overrides):** 允许用户用一个输入框填入自己的 OpenAI/Anthropic Key 以覆盖底层的默认免费网关配置 `models.json`。

## 五、 敏捷开发里程碑排期计划 (Phased Development Roadmap)

此计划为工程落地提供节奏控制，开发团队应逐一攻破各阶段的 Milestone。

### Phase 1: “凿穿底层通道” (MVP 最小可运行框架)
*   **攻坚重点:** Tauri 外壳和 Rust 侧代码的编写。
*   **关键交付:**
    *   构建无 UAC 烦恼的 `AppData` 目录解压策略。
    *   完成 Node.js Portable 的隐蔽下载机制及基于它的静默网络连通/ZIP源码拉取逻辑。
    *   最简陋的 HTML 前端（只有一个进度条和一个按钮），实现能调起 Node 服务进程。
*   **验收标准:** 一台从来没装过开发环境的虚拟机，双击应用，3 分钟后进程启动不闪退。

### Phase 2: “注入灵魂与颜值” (核心体验版本)
*   **攻坚重点:** 业务特性的“无感注入”以及现代极简 UI 的重写。
*   **关键交付:**
    *   脚本内自动生成并注入配置（免密 API Key 入口，以及 `models.json` 和 `openclaw.json` 的拼接映射）。
    *   随包打包 `skill_creator` 和 `skill-lookup` 至源码技能目录。
    *   开发精美的 React/Vue 前端 UI（状态大屏、快捷访问栏、向导弹窗）。
*   **验收标准:** 用户只需点击“继续”、“启动”，即可在被弹出的 Web 页面中与具有写代码能力的 AI 开始交谈。

### Phase 3: “商业化与稳定加固” (增强版管家)
*   **攻坚重点:** 错误收敛、容灾处理和用户体验打磨。
*   **关键交付:**
    *   **拦截器:** 实现杀毒软件白名单提示警告、实现端口冲突的探活检测。
    *   实现 System Tray 托盘静默运行逻辑及彻底清理缓存按钮。
    *   完善安装程序的封包（Inno Setup 添加防火墙白名单注册代码，签署 EV 数字证书申请准备）。
*   **验收标准:** 实现类似 Docker Desktop 般稳定、丝滑的本地服务体验，即便用户进行各种作死操作也能有一键修复或明晰的弹窗指引退路。
